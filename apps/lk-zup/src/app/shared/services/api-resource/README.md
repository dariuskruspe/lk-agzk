Система api-resources позволяет определять переиспользуемые ресурсы апи с возможностью настройки кеширование.

Кеширование происходит в indexed-db если доступно, или local-storage если нет.

Возможности:

- Блокировка запросов, чтобы не происходило состояния гонки когда один запрос вызван одновременно в нескольких местах.
- Поддержка динамических ключей
- Автоматическая типизация аргументов и типа результата
- Ресурс может возвращать как observable так и promise
- Ресурс может подключать глобальные сервисы используя inject()
- Поддержка режима использования с signals-api
