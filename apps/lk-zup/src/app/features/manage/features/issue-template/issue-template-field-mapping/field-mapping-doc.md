Поля:
name - строка - имя
nameOnFront - строка - имя на фронте
nameOnIssue - строка - имя в заявке
uniquenessControl - булево - контроль уникальности (будет ли проверяться поле на уникальность)
parent - число - Номер строки поля, которому принадлежит данная строка. Например, будет поле типа массив, называться, Сотрудники. Внутри массива будет поле Сотрудник. Полю сотрудник нужно будет присвоить номер строки родителя (порядковый номер).
id - строка - уникальная строка типа гуид
indicator - строка - показатель. (не нашел где используется данная строка, но в некоторых случаях она заполнена)
fileForSignature - булево - используется ли файл для подписания в данном поле.

onOtherEmployees - булево - доступно ли поле при оформлении на других сотрудников менеджером
onOtherEmployeesIsEmpField - булево - является ли это полем сотрудник при оформлении на других сотрудников
onApplicant - булево - доступно ли поле при оформлении на заявителя
onApplicantIsEmpField - булево - является ли это полем сотрудник при оформлении на заявителя

Пример структуры:
{
"onOtherEmployees": false,
"onApplicant": false,
"onOtherEmployeesIsEmpField": true,
"onApplicantIsEmpField": true,
"name": "СотрудникЗаявки",
"nameOnFront": "issueEmployee",
"nameOnIssue": "Сотрудник",
"uniquenessControl": false,
"parent": 0,
"id": "bae10205-b96e-418a-b4c5-6510daf50685",
"indicator": "",
"fileForSignature": false
}
